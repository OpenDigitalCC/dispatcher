Dispatcher — Architecture

Dual Dispatcher & Trust Model

Certificate-based trust · Pairing workflow · Allowlist enforcement

diagram 2 of 2 — dual dispatcher · shared CA · pairing workflow · allowlist enforcement

Trust Model

  • Single CA issues all certificates
  • Each dispatcher holds a signed cert
  • Agents only accept connections from verified dispatchers
  • Pairing is a one-time certificate exchange
  • No shared secrets — mTLS throughout

What Agents Do

  • Listen on port 7443 for incoming commands
  • Verify dispatcher cert against CA before accepting
  • Check script name against local allowlist
  • Execute script as dispatcher system user
  • Return stdout, stderr and exit code to dispatcher
  • Log all actions to syslog with structured fields

What Dispatchers Do

  • Accept commands from operator CLI or API
  • Fan commands out to agents in parallel
  • Aggregate and present results
  • Maintain agent registry with cert expiry
  • Run pairing mode to onboard new agents
  • Log all dispatches to syslog
